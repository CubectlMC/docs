# DNS

## Ответственность

DNS-операции реализуются в `instance-service`

DNS-записи создаются и удаляются вместе с инстансом, публичным портом и lifecycle

## Абстракция провайдера

Нужна общая архитектура DNS-провайдера:

```text
org.cubectl.instance.dns
  provider
  regru
```

Первой реализацией планируется REG.RU

Provider interface должен покрывать:

- Создание A-записи
- Создание SRV-записи
- Обновление SRV-записи
- Удаление записей инстанса
- Проверку текущего состояния записи

## A/SRV

Для подключения без ручного указания порта используется связка A и SRV

A-запись указывает домен на IP сервера

SRV-запись указывает порт конкретного инстанса

Админские настройки DNS в web UI пока не планируются

Конфигурация провайдера хранится в конфигурации сервиса и секретах окружения
