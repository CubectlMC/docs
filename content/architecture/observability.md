# Метрики и логи

## Метрики

`metric-service` собирает снимки runtime-информации каждые 15 секунд и сохраняет их в PostgreSQL

Собираемые данные:

- CPU
- RAM
- Disk
- Network
- Uptime
- Container status
- Timestamp снимка

Исторические данные сохраняются для будущих графиков

В MVP графики не строятся

REST отдает текущий snapshot при открытии экрана инстанса

SSE отдает новые snapshot только пока пользователь находится на экране runtime-информации

Один backend collector опрашивает Docker API централизованно

Количество открытых UI-клиентов не должно увеличивать число Docker stats запросов

## Логи

Живые Docker-логи отдаются через SSE

Ответственность за живые логи находится в `instance-service`, потому что источник данных — поток Docker stdout/stderr

Файловые логи не входят в MVP как отдельный API
