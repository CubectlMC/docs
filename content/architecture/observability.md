# Метрики и логи

## Метрики

`metric-service` собирает снимки метрик каждые 15 секунд и сохраняет их в PostgreSQL

Собираемые данные:

- CPU
- RAM
- Disk
- Network
- Uptime
- Container status
- Timestamp снимка

Исторические данные выдаются через REST

Live-обновления выдаются через SSE после первичной загрузки истории

## Логи

Живые Docker-логи отдаются через SSE

Ответственность за живые логи находится в `instance-service`, потому что источник данных — поток Docker stdout/stderr

Файловые логи не входят в MVP как отдельный API
