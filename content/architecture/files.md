# Файлы и внешние каталоги

## Ответственность file-service

`file-service` отвечает за каталог контента и подготовку файлов для инстансов

В MVP нет generic file upload API

## Контент

Поддерживаемые типы контента:

- Моды
- Плагины
- Дата-паки

Ресурспаки, конфигурационные файлы, backups и generic upload исключены из MVP

## Провайдеры контента

Поиск и скачивание контента строится через общую архитектуру провайдеров

Доменные пакеты:

```text
org.cubectl.file.provider.modrinth
org.cubectl.file.provider.curseforge
```

Modrinth:

- Поиск проектов
- Карточка проекта
- Версии проекта
- Зависимости
- Скачивание jar с CDN

CurseForge:

- Fallback-поиск
- Metadata файлов
- Зависимости
- Скачивание при доступности файла через API

## Модель выбора контента

Контент выбирается через provider id, project id и version id

`file-service` скачивает и подготавливает файлы внутренне

`instance-service` хранит выбранный набор контента в модели инстанса

Изменение контента разрешено только когда инстанс остановлен

## Обновление контента

`file-service` проверяет актуальные версии через внешних провайдеров

Результаты проверки хранятся в `content_update_checks`

Нажатие кнопки обновления создает запись в `content_update_actions`

Применение обновления разрешено только когда инстанс остановлен
